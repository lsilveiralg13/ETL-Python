# Data Automation & ETL Pipeline 🚀

Este repositório contém uma estrutura robusta para fluxos de **ETL (Extração, Transformação e Carga)** e automação de processos utilizando Python e SQL. O projeto foi desenvolvido com foco em escalabilidade, modularização e eficiência no processamento de dados.

## 📋 Visão Geral
O objetivo deste projeto é fornecer ferramentas automatizadas para a manipulação de grandes volumes de dados, garantindo a integridade das informações desde a fonte original até o destino final (Data Warehouse ou Bancos de Dados Relacionais).

## 🛠️ Stack Técnica
* **Python**: Core do processamento e automação de rotinas.
* **Manipulação de Dados**: Uso de bibliotecas de alta performance para tratamento de dados estruturados.
* **SQL**: Scripts otimizados para persistência e consulta de dados.
* **Git**: Versionamento seguindo boas práticas de organização e exclusão de arquivos binários/temporários.

## 📂 Organização do Repositório
* `scripts/`: Lógica de automação e processamento de dados.
* `sql/`: Scripts de definição de esquema e consultas analíticas.
* `.gitignore`: Configuração rigorosa para garantir a segurança e leveza do repositório, impedindo o versionamento de dados sensíveis ou arquivos temporários.

## ⚙️ Diferenciais Técnicos
* **Processamento Otimizado**: Scripts preparados para lidar com diferentes formatos de entrada (Parquet, CSV, Excel).
* **Segurança de Dados**: Estrutura preparada para ambientes produtivos, separando lógica de execução de arquivos de dados.
* **Manutenibilidade**: Código organizado de forma modular para facilitar a implementação de novas regras de negócio.
